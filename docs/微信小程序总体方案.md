# 乐魂鸣霄引微信小程序总体方案

## 一、项目定位

《乐魂鸣霄引》是一个以**盛唐文化**、**传统乐器拟人化**与**原创剧情叙事**为核心的国风 IP 项目。项目围绕古琴、琵琶、阮、二胡、埙、竖笛、筑、篪等乐器化灵角色展开，通过角色设定、视觉资产、剧情文本与视听内容共同构成“文化 IP + 数字传播 + 轻交互体验”的展示体系。

当前微信小程序的核心目标并不是建设重型社区或商业交易平台，而是优先完成一个**适合赛事展示、品牌传播、合作洽谈与内容承载**的轻量化产品入口。基于这一目标，小程序被定义为一个以 IP 展示为中心、以剧情与视频内容为延伸、以合作转化为补充的数字化窗口。

## 二、目标用户与核心价值

从仓库现有内容、项目文档和 Web 端演示结构来看，目标用户主要分为三类：国风动漫爱好者、赛事评委与投资人，以及高校或社区合作方。前者关注角色魅力、剧情设定和视觉风格；中间群体更关心项目完整度、可传播性和商业化潜力；后者则需要快速理解合作模式、活动形式与团队能力。

| 用户类型 | 关注重点 | 小程序承载价值 |
|---|---|---|
| 国风动漫爱好者 | 角色设定、立绘、剧情、视频内容 | 提升兴趣与分享传播 |
| 赛事评委与投资人 | 项目完整度、IP 资产、展示效果 | 快速建立项目认知 |
| 高校与社区合作方 | 合作模式、课程活动、联系方式 | 支撑留资与合作转化 |

## 三、产品目标

小程序首期应服务四个明确目标。第一，是让用户在最短时间内理解“乐魂鸣霄引”这一 IP 的世界观与角色体系。第二，是将已有的角色立绘、剧情文本、视频内容系统化整理为统一入口。第三，是为比赛答辩、项目路演和对外展示提供稳定、清晰、可分享的数字载体。第四，是通过合作预约与联系方式模块，建立基础转化通道。

因此，产品策略应坚持“**重展示、轻交互、可迭代**”。展示内容优先保证完成度，互动功能则采用逐步补充的方式，以避免在早期阶段投入过多开发成本。

## 四、信息架构

结合现有 Web MVP 的页面组织，微信小程序建议采用五个一级导航页面。

| 一级模块 | 主要内容 | 首期建议 |
|---|---|---|
| 首页 | Banner、项目简介、四大入口、角色速览、合作导流 | 必做 |
| 乐魂 IP 馆 | 群像展示、角色卡片、角色详情页 | 必做 |
| 声影剧场 | 动画样片、幕后花絮、CV 内容、视频播放 | 必做 |
| 鸣霄剧本阁 | 世界观导读、剧情节选、创作手记 | 第二阶段 |
| 我的 | 收藏、浏览记录、合作预约、关于我们 | 必做 |

在页面层级上，建议首页承担总入口作用，角色详情页和剧情内容页作为深度承载页面，个人中心则作为功能汇总与转化页面使用。

## 五、核心模块设计

### 5.1 首页

首页承担项目总览与内容导流功能。页面应通过大幅视觉 Banner 快速建立第一印象，并清晰展示“原创 IP + 盛唐文化 + CV 演绎”的核心卖点。首页同时需要承担转化入口作用，因此除内容分发外，还应设置合作预约、外部账号关注与活动入口。

| 功能区块 | 主要内容 |
|---|---|
| 顶部 Banner | 海报、动画封面、剧本宣传图 |
| 核心入口 | IP 馆、剧场、剧本阁、互动体验坊 |
| 项目简介 | 100 字左右项目简介与亮点概述 |
| 角色速览 | 重点角色卡片与快捷跳转 |
| 底部快捷入口 | 合作预约、社媒跳转、团队介绍 |

### 5.2 乐魂 IP 馆

乐魂 IP 馆是整个产品最核心的资产展示区。该模块应重点承载角色立绘、背景故事、能力设定、服饰纹样和文化细节解读。页面形态上建议采用“全景群像 + 角色卡片 + 详情页”的三级组织方式，以兼顾浏览效率和内容深度。

| 角色 | 乐器 | 定位 | 武器/能力 |
|---|---|---|---|
| 谢云止 | 古琴 | 谋士 / 太学博士 | 太极两仪棋 |
| 薛枕月 | 琵琶 | 将门女将 | 九环斩月刀 |
| 阮泠 | 阮 | 治愈者 | 慈烬符 |
| 麦勒 | 二胡 | 草原少年 | 弦幻弓 |
| 苏冥幽 | 埙 | 侠盗 | 泣血翎 |
| 柳瑶琮 | 竖笛 | 情报掌控者 | 错金银骰子 |
| 裴承钧 | 筑 | 反派 / 大理寺卿 | 音律控魂术 |
| 苏照野 | 篪 | 亦正亦邪 | 缚灵链 |

首期上线建议优先突出谢云止、薛枕月、麦勒三位代表角色，以便尽快形成完整展示闭环。

### 5.3 声影剧场

声影剧场负责承载项目的动态内容，包括动画样片、角色展示片、幕后创作花絮与 CV 配音相关内容。该模块一方面能够增强项目的表现力，另一方面也是评委和合作方快速判断团队执行力的重要窗口。

首期不建议引入复杂评论系统，可以先保留播放、点赞、收藏与分享等轻量能力。待后续内容产能稳定后，再考虑更深层的互动设计。

### 5.4 鸣霄剧本阁

剧本阁用于展示世界观背景、剧情高光节选与创作手记，是项目“原创性”和“文化深度”的核心表达区域。通过图文结合的方式，可让用户在较短时间内理解剧情张力与设定逻辑。

该模块建议在第二阶段重点完善。首发版本只需上线世界观导读与 2 至 3 个高光片段即可，不必一开始就放出过多剧本文本。

### 5.5 我的与合作转化

个人中心不需要复杂化，而应优先承担收藏、浏览记录、合作预约和关于我们等基础功能。尤其是合作预约，应设计为极简表单，以便在赛事展示、项目推介和外部合作场景中实现留资。

| 功能项 | 作用 |
|---|---|
| 收藏内容 | 沉淀用户偏好 |
| 浏览记录 | 方便回访内容 |
| 合作预约 | 收集合作线索 |
| 关于我们 | 展示团队信息与联系方式 |
| 版权声明 | 体现原创规范性 |

## 六、视觉与内容规范

整体视觉建议保持“盛唐、皮影、国风、器灵”四个关键词的统一表达。色彩上可延续朱红、墨黑、鎏金与宣纸白的组合，以形成较强的品牌识别。标题字体应偏衬线化，正文则保持清晰易读。边框、分割线和卡片装饰可适度引入皮影镂空纹样、宝相花、卷草纹与忍冬纹元素，但要控制密度，避免页面过于繁复。

内容上则建议保持统一的卡片结构与叙事模板，例如角色详情页统一包含“身份定位、背景故事、能力设计、服饰纹样、彩蛋补充”五个部分，以方便后续增量维护。

## 七、开发实施路线

结合当前仓库中的 Web MVP，建议将后续建设分为三个阶段。第一阶段完成展示型 MVP，第二阶段补充叙事内容，第三阶段再增加轻互动与转化功能。

| 阶段 | 周期 | 目标 | 主要内容 |
|---|---|---|---|
| 第一阶段 | 2–3 周 | 搭建展示型 MVP | 首页、IP 馆、剧场、我的 |
| 第二阶段 | 第 4–6 周 | 补齐内容深度 | 全角色档案、剧本阁、更多视频 |
| 第三阶段 | 第 7–10 周 | 增加互动与转化 | 互动体验坊、收藏、预约、运营机制 |

## 八、技术路线建议

考虑到当前仓库已有 React + Vite + TypeScript 的 Web 端演示基础，后续如果要正式开发微信小程序，推荐优先选择 **Taro + React + TypeScript** 作为技术路线。这样可以最大程度延续当前页面结构、组件拆分习惯与前端协作方式，降低迁移成本。

如果只是为了比赛或演示快速上线，当前仓库中的 Web 静态版也可以继续作为对外展示站存在，再并行推进小程序 MVP 的开发。这样既能保证短期可展示，又能为后续正式产品保留演进空间。

## 九、仓库文档整理原则

当前仓库中原有多份小程序规划文档存在明显的主题重叠，分别从定位分析、MVP 设计和功能规格三个角度展开。为提高仓库可维护性，现阶段建议以本文件作为**统一主文档**，其余旧文档转入 `docs/archive/` 保存历史版本。这样既保留原始资料，也避免顶层目录过于分散。

整理后建议仓库采用如下结构：

```text
piying/
├── README.md
├── assets/
├── docs/
│   ├── 微信小程序总体方案.md
│   └── archive/

│   ├── client/
│   ├── server/
│   └── package.json
```

这一结构的核心原则是：**顶层保留入口说明，源码与素材分层存放，规划文档集中到 docs，旧文档归档而不散落。**
